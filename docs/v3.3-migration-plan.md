# v3.3 Slice A Migration Plan

**Date**: 2026-04-16
**Depends on**: Debate 016 / 017 / 018 synthesis
**Scope**: Schema hardening — scope enum + origin CHECK + length CHECK + recall_miss_log + compost_cache DDL

---

## 决策摘要

### 来自 Debate 016
- Engram 与 Compost 边界从"字数"改为"SLO + 来源"
- schema 层锁死关键不变量（非文档约定）
- v4-v7 全砍，Engram 稳定在 v3.x

### 来自 Debate 017
- Migration 必须 `BEGIN IMMEDIATE ... COMMIT` 单事务
- FTS5 rebuild 必须显式 `INSERT INTO memories_fts(memories_fts) VALUES('rebuild')`
- WAL checkpoint 不在 migration 中间做
- 删除 LoC 预算，改为功能验收清单

### 来自 Debate 018
- `length(content) <= 4000`（非 2000）— 基于 SQLite 4KB page size 物理边界
- kind-specific length 不在 schema 层，在应用层 warning + lint
- Schema CHECK = 数据完整性红线，不是风格偏好

### 来自后续补丁（用户提出）
- unpin API 加入 v3.3（由 debate 017 的 2:1 REJECT 修订为 SUPPORT，理由写入 `017-amendment.md`）
- unpin API 第一个实战 = dogfood 清 v1 pinned 4 条（15e2/b319/5bfe/66a4）

## Migration 执行步骤

### Step 1: Unpin API 实现（前置）
位置: `src/engram/store.py`
- `MemoryStore.unpin(memory_id: str) -> dict` — 单条，ops_log 记录
- `engram unpin <id>` CLI，带 `--confirm` 和 `--yes` 选项
- `mcp__engram__unpin` MCP tool
- 3 tests: 成功 / 不存在 id / 已非 pinned

### Step 2: Dogfood
运行顺序:
1. `engram unpin 15e2bad36425 --yes`
2. `engram forget 15e2bad36425`
3. 重复 b3190c88f62b / 5bfe29263fd0 / 66a41d866025
4. `engram recall project=compost query=final-form` 验证只剩 v2（3a04dbe732e6）

### Step 3: DB Backup
```bash
sqlite3 ~/.engram/engram.db ".backup ~/.engram/engram.db.pre-v3.3"
```

### Step 4: 运行 Migration SQL
见 `src/engram/migrations/001_slice_a_schema.sql`。

通过 `engram migrate --slice=a --dry-run` 先看报告，然后 `engram migrate --slice=a --apply`。

### Step 5: 验证
按功能验收清单逐条检查（见下文）。

---

## Rollback 方案

如果 Step 4 或 Step 5 失败:
```bash
# SQL 事务未 commit → 自动回滚 (BEGIN IMMEDIATE ... COMMIT)
# 如果已 commit 但验证失败:
mv ~/.engram/engram.db ~/.engram/engram.db.broken
cp ~/.engram/engram.db.pre-v3.3 ~/.engram/engram.db
```

Migration SQL 本身在 `BEGIN IMMEDIATE` 事务内，任何 step 失败 → 整个回滚到 pre-migration 状态。

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| FTS5 rebuild 后搜索静默失败 | 验证清单显式跑 `engram search` 测试 |
| rowid 漂移影响 recall ranking | INSERT 时显式保留 rowid |
| WAL 未 checkpoint 导致 DB 文件膨胀 | 迁移 commit 后手动 `PRAGMA wal_checkpoint(TRUNCATE)` |
| obsolete 记忆被 CHECK 拒绝 | audit 确认无违规，全部迁移 |
| agent 中途写入导致事务冲突 | BEGIN IMMEDIATE 锁定 writer，无并发问题 |
| dogfood v1 pinned 清理失败 | Step 2 失败时停止，修复 unpin API 再继续 |
